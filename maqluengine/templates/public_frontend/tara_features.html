Features
One stop shop of archaeology project needs
  • Backend repository of non-media data(Reposit)
  • Can link and display outside media(DAMs)
  • Automated front-end(Publish) to replace a separate “Project website” that’s integrated with the database
  • Tools to analyze, import, and export the data that’s in it
  • Can be used for work in the lab and in the field with stable data connections.
Online Web-application built on top of Django/MySQL/OpenLayers/Chart/JQuery
  • Incredibly Generic– but with intended constraints
  • Designed to outwardly mimic existing legacy databases, but drastically different under the hood
  • Pro: Incredibly dynamic with few tables needed to define (any?) data types == less expensive SQL table joins
  • Con: Will never be as ‘fast’ as a customized, highly bespoke SQL design, because of larger tables
Data Export / Import Tools
  • Export the entire project or form sets as CSV or JSON with all relations intact
  • Import pre-existing digitized data sets from CSV files
  • Import geoJSON data as a new populated FormType(Entity Type) with all attached metadata and vector information
Optional public facing webpage and blog Manager
  • Integrates an auto-generated website for the project linked to the same database with the ability to customize a menu and add new pages and content
  • Integrates a simple blog for each project, also tied to the same database
  • Removes the need for an external public-facing website, and keeps it linked to your existing data
Templating System for designing Form layouts
  • Allows project admins to design layouts for their entity types with a variety of different widgets to represent the data
  • Relational Fields can be shown as dropdown lists, image gallery viewers, and thumbnailed lists
  • Text based fields can be resized as needed
  • All fields can be positioned and saved
  • Future Updates will allow text-based fields to be format enforced, e.g. boolean check boxes, numerical
Recycling Bin System for Safe Deletion and Restoration of Data
  • Safely delete data and restore it
  • Data is only stored in a temporary 'recycling bin' where high-level project access is required to permanently delete the data
  • Restored data is immediately put back exactly--with all previous relationships, and meta intact
Highly Malleable Entity and Field Types
  • Record Attribute Types, e.g. text-based fields can quickly converted into relation fields, and automated lookups will search for any matching relations of the chosen entity type
  • Record Reference Types, e.g. relation fields, can quickly be converted in to text-based fields
  • Record Attribute Types can be interchanged with a Form Type's Form IDs at will
  • All fields, entity types, and forms can be renamed at will without affecting the database or its relationships
Can link to outside media servers and display various media formats
  • project admins can enter in URIs to an external media server or file server that hosts their media/image content
  • These uris are automatically linked to created media Formtypes with matching form IDs
  • Linkable media that can be viewed: PDF's, Web-friendly Image formats(.png,.jpg,etc), and 3D .obj files
Privacy and Security Restrictions
  • Project can set public/private access to various levels of data
  • Internal Project user security on an easy 5 stage access level system of data manipulation/creation
  • Currently working on getting longterm HTTPS support for additional external security
  • Leverages Django's existing CSFR security and sessioning
  • Any security flags raised in the system and blocked are immediately logged on the server logs as well as in a database log accessible quickly by the network admin
Geospatial Engine for Vector data
  • Can handle large sets of vector data in the geoJSON format
  • All Formtypes in the system have their own mini GIS interface to manually draw in vector shapes for the Form
  • Both the larger and mini-engine can support viewing multiple layers
  • Basic symbology can be applied to any geospatial vector data--utilizing the linked and related meta data to filter that symbology
  • Ability to manipulate the colors and transperencies of symbology manually, or automatically through randomly generated gradients
  • Multiple allowed projection systems using GDAL
  • base map layers provided freely by ESRI
  • Ability to click on vector features to bring up feature/form attributes
Responsive user Feedback for longer server processes
  • all longer processing provide progress bars with asynchronously updated information to user
  • important changes alert the user of a successful 'save' or change of data
